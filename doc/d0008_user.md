## 2. 설치 가이드

### 2.1 설치 실행

1. `StokAI_Setup_0.1.0.exe` 실행
2. 설치 마법사의 안내를 따라 진행
3. 설치 옵션 선택:
   - **바탕화면 바로가기**: 바탕화면에 아이콘 생성
   - **방화벽 규칙 추가**: 웹 대시보드 외부 접속용 (권장)
   - **Defender 제외 등록**: 실행 차단 방지 (권장)

### 2.2 Windows 보안 차단 해결

StokAI는 코드 서명이 되지 않은 프로그램이므로 보안 차단이 발생할 수 있습니다.

#### SmartScreen 차단 시

1. "Windows가 PC를 보호했습니다" 팝업이 뜨면
2. **"추가 정보"** 클릭
3. **"실행"** 버튼 클릭

#### 오류 코드 4551 (WDAC/AppLocker 차단)

조직의 앱 제어 정책이 차단하는 경우입니다.

**방법 1: 파일 속성에서 차단 해제**
1. `StokAI.exe` 우클릭 > **속성**
2. 하단 **"차단 해제"** 체크 > **확인**

**방법 2: PowerShell (관리자 권한)**
```powershell
# Defender 제외 등록
Add-MpPreference -ExclusionPath "C:\Program Files\StokAI"

# SmartScreen 차단 해제
Get-ChildItem -Path "C:\Program Files\StokAI" -Recurse | Unblock-File
```

**방법 3: 로컬 그룹 정책 편집기 (조직 정책인 경우)**
1. `Win+R` > `gpedit.msc` 실행
2. 컴퓨터 구성 > Windows 설정 > 보안 설정 > 응용 프로그램 제어 정책
3. AppLocker > 실행 파일 규칙 > 새 규칙 만들기
4. StokAI 설치 경로(`C:\Program Files\StokAI`)를 허용으로 추가

## 4. 웹 대시보드

### 4.1 개요

StokAI는 외부에서 모니터링/제어할 수 있는 웹 대시보드를 내장하고 있습니다.

| 항목 | 값 |
|------|---|
| 기본 포트 | 8080 |
| 기본 아이디 | admin |
| 기본 비밀번호 | stokai2026 |
| 프로토콜 | HTTP |
| 인증 | Bearer Token (세션 기반) |

### 4.2 접속 방법

프로그램 실행 시 상태바에 웹 대시보드 URL이 표시됩니다.

- **같은 PC**: `http://localhost:8080`
- **같은 네트워크(LAN)**: `http://192.168.x.x:8080`
- **외부(인터넷)**: `http://공인IP:8080` (포트포워딩 필요)

### 4.3 웹 대시보드 기능

| 기능 | API | 설명 |
|------|-----|------|
| 로그인/로그아웃 | POST /api/login, /api/logout | 세션 토큰 발급/해제 |
| 전체 상태 조회 | GET /api/status | 계좌, 포지션, AI 시그널 등 |
| 계좌 정보 | GET /api/account | 예수금, 총자산 |
| 보유 종목 | GET /api/positions | 포지션 목록 |
| 매매 로그 | GET /api/trade-logs | 최근 매매 이력 |
| 알림 목록 | GET /api/alerts | 시스템/매매 알림 |
| AI 시그널 | GET /api/ai-signal | 현재 AI 분석 결과 |
| 시장 감성 | GET /api/sentiment | 뉴스 감성분석 현황 |
| 시장 지수 | GET /api/market | KOSPI/KOSDAQ 지수 |
| 자동매매 시작 | POST /api/autotrade/start | 전략 지정 시작 |
| 자동매매 중지 | POST /api/autotrade/stop | 자동매매 중단 |
| 킬 스위치 ON | POST /api/kill-switch/on | 비상 정지 |
| 킬 스위치 OFF | POST /api/kill-switch/off | 비상 정지 해제 |
| 실시간 업데이트 | WebSocket /ws | 1초 주기 상태 동기화 |

## 5. 포트포워딩 설정 가이드

외부 인터넷에서 웹 대시보드에 접속하려면 **공유기(라우터) 포트포워딩** 설정이 필요합니다.

### 5.1 사전 확인

```
내부 IP 확인 방법:
1. Win+R > cmd 실행
2. ipconfig 입력
3. "IPv4 주소" 확인 (예: 192.168.0.10)

공인 IP 확인:
- StokAI 실행 시 상태바에 표시됨
- 또는 브라우저에서 https://api.ipify.org 접속
```

### 5.2 공유기별 포트포워딩 설정

#### ipTIME 공유기 (가장 일반적)

1. 브라우저에서 `http://192.168.0.1` 접속 (관리자 로그인)
2. **관리도구** > **고급 설정** > **NAT/라우터 관리** > **포트포워드 설정**
3. 아래 정보 입력:

| 항목 | 값 |
|------|---|
| 규칙이름 | StokAI |
| 내부 IP 주소 | 192.168.0.XX (StokAI PC의 내부 IP) |
| 프로토콜 | TCP |
| 외부 포트 | 8080 |
| 내부 포트 | 8080 |

4. **추가** > **적용**

#### KT 공유기 (GiGA WiFi 등)

1. `http://172.30.1.254` 접속
2. **장치설정** > **트래픽 관리** > **포트 포워딩**
3. 외부 포트: 8080, 내부 IP: PC IP, 내부 포트: 8080, TCP
4. **추가** > **저장**

#### SK 공유기 (BroadBand)

1. `http://192.168.25.1` 접속
2. **고급설정** > **NAT 설정** > **포트 포워딩**
3. 서비스 포트: 8080, IP 주소: PC IP, 포트: 8080, TCP
4. **적용**

#### LG U+ 공유기

1. `http://192.168.219.1` 접속
2. **네트워크 설정** > **NAT 설정** > **포트포워딩**
3. 외부 포트: 8080, 내부 IP: PC IP, 내부 포트: 8080, TCP
4. **저장**

#### ASUS 공유기

1. `http://192.168.1.1` 접속
2. **WAN** > **Virtual Server / Port Forwarding**
3. Service Name: StokAI, Port Range: 8080, Local IP: PC IP, Local Port: 8080, Protocol: TCP
4. **Apply**

### 5.3 포트포워딩 확인

설정 후 외부 접속 확인:

```
방법 1: 스마트폰 (WiFi 끄고 LTE/5G로 접속)
  - 브라우저에서 http://공인IP:8080 접속

방법 2: 외부 포트 확인 사이트
  - https://www.yougetsignal.com/tools/open-ports/
  - Remote Address: 공인IP, Port Number: 8080 > Check 클릭
```

### 5.4 보안 주의사항

| 항목 | 권장 설정 | 설명 |
|------|----------|------|
| 기본 비밀번호 변경 | 필수 | config.json에서 web_dashboard.password 변경 |
| 포트 변경 | 권장 | 8080 대신 비표준 포트 사용 (예: 18080) |
| 접속 제한 | 권장 | 공유기에서 특정 IP만 허용 설정 |
| HTTPS | 향후 지원 | 현재 HTTP만 지원, 공공 네트워크 사용 시 주의 |

### 5.5 고정 IP 설정 (권장)

DHCP로 내부 IP가 변경되면 포트포워딩이 끊깁니다. PC의 IP를 고정하세요:

1. **설정** > **네트워크 및 인터넷** > **이더넷** > **IP 할당 편집**
2. **수동** 선택
3. 아래 정보 입력:

| 항목 | 예시 값 |
|------|--------|
| IP 주소 | 192.168.0.100 |
| 서브넷 마스크 | 255.255.255.0 |
| 게이트웨이 | 192.168.0.1 |
| DNS | 8.8.8.8, 8.8.4.4 |

### 5.6 DDNS 설정 (공인 IP가 변경되는 경우)

가정용 인터넷은 공인 IP가 주기적으로 변경됩니다. DDNS를 사용하면 도메인으로 접속할 수 있습니다.

#### ipTIME DDNS (무료)
1. 공유기 관리페이지 > **고급 설정** > **특수 기능** > **DDNS 설정**
2. 호스트이름: `mystockai` > 등록
3. 접속: `http://mystockai.iptime.org:8080`

#### No-IP DDNS (무료)
1. https://www.noip.com/ 회원가입
2. 호스트이름 등록 (예: `mystockai.ddns.net`)
3. No-IP DUC 클라이언트 설치 (IP 자동 갱신)
4. 접속: `http://mystockai.ddns.net:8080`

## 9. 문제 해결 (FAQ)

### Q1. 프로그램이 실행되지 않습니다 (오류 4551)
> 섹션 2.2 "오류 코드 4551" 해결 방법을 참고하세요.

### Q2. 웹 대시보드에 외부에서 접속이 안됩니다
> 1. 방화벽 규칙이 등록되었는지 확인 (제어판 > Windows Defender 방화벽 > 고급 설정)
> 2. 공유기 포트포워딩이 설정되었는지 확인 (섹션 5 참고)
> 3. 공인 IP가 변경되지 않았는지 확인

### Q3. 키움 API 연결이 안됩니다
> 1. 키움증권 Open API+ 신청 완료 여부 확인
> 2. 영웅문 HTS가 동시 실행 중이면 종료 (동시 접속 1대 제한)
> 3. 32bit Python 브릿지 프로세스 확인

### Q4. AI 분석이 작동하지 않습니다
> 1. AI 설정에서 API Key가 올바른지 확인
> 2. OpenAI/DeepSeek 계정 잔액 확인
> 3. 네트워크 연결 확인

### Q5. 킬 스위치가 자동 발동되었습니다
> MDD(최대낙폭)가 15%를 초과하면 자동 발동됩니다.
> 수동으로만 해제할 수 있으며, 원인 분석 후 해제하세요.
