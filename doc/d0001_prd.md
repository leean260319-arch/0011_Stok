## 2. 프로젝트 개요

### 2.1 목적

키움증권 Open API+와 AI 기술을 결합하여, 개인 투자자가 **뉴스 감성분석 + 기술적 차트 분석**을 기반으로 자동 매수/매도를 수행할 수 있는 데스크탑 프로그램을 개발한다.

**핵심 목표:**

1. 키움증권 API를 통한 실시간 시세 조회 및 자동 주문 실행
2. AI(LLM) 기반 뉴스 감성분석으로 시장 심리 파악
3. 기술적 지표(RSI, MACD, 볼린저 밴드 등) 기반 차트 분석
4. AI 분석 결과를 종합한 자동 매매 전략 실행
5. 다양한 대시보드를 통한 포트폴리오/실적/시장 현황 모니터링
6. 프로그램 재시작 시 이전 데이터 완전 복원 (세션 지속성)
7. 개인정보 암호화 로컬 저장 (배포 시 각 PC에 독립 저장)

### 2.2 배경

#### 문제 상황

- 개인 투자자의 감정적 매매로 인한 손실 발생
- 뉴스/공시 정보를 수동으로 확인하는 시간 부담
- 기술적 분석 지표를 실시간으로 모니터링하기 어려움
- 매매 타이밍을 놓치는 경우 빈번
- 기존 자동매매 프로그램은 AI 분석 기능 부재 또는 고비용

#### 기술적 배경

- 키움증권 Open API+: 국내 개인투자자 대상 가장 보편적인 자동매매 API
- LLM 기술 발전: GPT-4o-mini 등 저비용 고성능 클라우드 모델 등장
- 클라우드 LLM 구독: GPT-4o-mini 월 $0.23 수준의 초저비용 감성분석
- Python 생태계: pandas-ta, lightweight-charts 등 금융 분석 도구 풍부

### 2.3 범위

#### In-Scope

- 키움증권 Open API+ 연동 (실시간 시세, 주문, 잔고 조회)
- AI 기반 뉴스 감성분석 (클라우드 LLM API)
- 기술적 지표 기반 차트 분석 (150+ 지표)
- 자동 매수/매도 전략 실행
- 실시간 포트폴리오 대시보드
- 매매 이력/수익률 분석 대시보드
- 뉴스 감성 트렌드 대시보드
- 백테스팅 (과거 데이터 전략 검증)
- 로컬 암호화 DB (SQLCipher)
- 프로그램 재시작 시 데이터 복원
- 데스크탑 앱 배포 (PyInstaller)
- 리스크 관리 (손절/익절, 일일 손실 한도, 킬 스위치)

#### Out-of-Scope

- 해외 주식 거래 (1차 버전)
- 선물/옵션 거래 (1차 버전)
- 웹/모바일 앱 개발
- 다중 사용자/서버 기반 서비스
- HFT(초고빈도 매매)
- 자체 딥러닝 주가 예측 모델 학습 (1차 버전)
- 코인/암호화폐 거래

### 3.2 키움 API 32bit/64bit 브릿지 구조

```
[64bit Python - 메인 앱]          [32bit Python - API 브릿지]
+------------------------+        +------------------------+
| PyQt5/6 UI             |        | 키움 Open API+ OCX     |
| AI 분석 엔진           |  gRPC  | pykiwoom 래퍼           |
| 전략 엔진              | <----> | 실시간 시세 수신         |

| 64bit 전용 라이브러리   |        | 잔고/체결 조회           |
+------------------------+        +------------------------+
         (KOAPY gRPC 브릿지 활용)
```

**선택지:**
- **방법 A (권장)**: KOAPY gRPC 브릿지 - 64bit 메인 앱에서 32bit API를 gRPC로 호출
- **방법 B**: 전체 32bit Python 환경 - 단순하지만 일부 AI 라이브러리 제약
- **방법 C**: 소켓 통신 - 32bit/64bit 프로세스 간 TCP/파이프 통신

### 8.5 AI 종합 점수 산출

```
종합 점수 = (감성 점수 x 감성 가중치) + (기술 점수 x 기술 가중치)

기본 가중치:
  - 감성 가중치: 0.3 (30%)
  - 기술 가중치: 0.7 (70%)

기술 점수 계산:
  - RSI 시그널: +1(과매도 반등) / -1(과매수 하락) / 0(중립)
  - MACD 시그널: +1(골든크로스) / -1(데드크로스) / 0
  - 볼린저 밴드: +1(하단 터치 반등) / -1(상단 이탈) / 0
  - 거래량 급증: +0.5(양봉) / -0.5(음봉) / 0
  - 이동평균 정배열: +1 / 역배열: -1 / 0

기술 점수 = 각 시그널 합계를 -1.0 ~ +1.0으로 정규화

매매 기준:
  - 종합 점수 >= +0.6: 강한 매수 시그널
  - 종합 점수 >= +0.3: 약한 매수 시그널
  - 종합 점수 <= -0.6: 강한 매도 시그널
  - 종합 점수 <= -0.3: 약한 매도 시그널
  - -0.3 ~ +0.3: 관망
```

### 12.1 API 제약사항

| 항목 | 제한 |
|------|------|
| API 방식 | OCX (32bit COM 컨트롤) |
| 지원 OS | Windows 전용 |
| 초당 요청 | 최대 5회 |
| 시간당 요청 | 최대 1,000회 |
| 동시 접속 | 동일 ID 1대만 (중복 로그인 불가) |
| 모의/실 동시 | 불가 (하나만 선택) |
| 3개월 미접속 | 서비스 자동 해지 |

### 12.2 API 호출 스로틀링

```python
# 초당 5회 제한 준수를 위한 스로틀링
import time
from collections import deque

class APIThrottler:
    def __init__(self, max_calls=5, period=1.0):
        self.max_calls = max_calls
        self.period = period
        self.calls = deque()

    def wait_if_needed(self):
        now = time.time()
        # 오래된 호출 기록 제거
        while self.calls and self.calls[0] < now - self.period:
            self.calls.popleft()
        # 제한 초과 시 대기
        if len(self.calls) >= self.max_calls:
            sleep_time = self.calls[0] + self.period - now
            time.sleep(sleep_time)
        self.calls.append(time.time())
```

### 12.3 연동 방식 선택

| 방식 | 장점 | 단점 | 권장 |
|------|------|------|------|
| KOAPY (gRPC) | 64bit 호환, 프로덕션 적합 | 설정 복잡 | 권장 (기본) |
| pykiwoom (32bit) | 단순, 문서 풍부 | 64bit 라이브러리 제한 | 대안 |
| 소켓 통신 | 유연한 구조 | 직접 구현 필요 | 특수 경우 |
